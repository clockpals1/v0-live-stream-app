feat(operator): rebuild operator surface as a full Live Control Room

The operator (super-user) page used to be a separate, simpler card
stack -- it owned its own overlay/ticker push functions and didn't
have Branding, Scenes, Guests rail, the new Media deck (slideshow +
short video clip), or the Producer Deck switcher at all. Operators
managing a host's stream had a strictly worse experience than the
host themselves.

This commit rebuilds the operator interface to use the same Live
Control Room layout as the owner, sharing every deck and rail.

KEY DESIGN DECISIONS
====================

Single source of truth: useControlRoomState
-------------------------------------------
The operator now mounts the same hook the owner uses. Every setter
in that hook does two things:

  1. UPDATE streams row (overlay / ticker / branding / scenes / clip)
  2. .send() the matching broadcast event over the chat channel

Both writes are scoped by streamId, so an owner and an operator
editing the same stream never fork state. Viewers receive identical
events regardless of who clicked. Last-write-wins on the row matches
the same pattern we already use for cohost takeover.

Read-only program preview via useSimpleStream
---------------------------------------------
The operator does NOT mount useHostStream. They have no business
holding peer connections to viewers; an accidental mount could yank
tracks from the live broadcast. Instead they watch the same output
every viewer sees through useSimpleStream and we composite the live
overlay + ticker on top so the operator's preview is faithful to
what's actually on screen.

Owner's plan (not operator's plan) gates Branding
-------------------------------------------------
app/host/stream/[roomCode]/page.tsx now resolves the OWNER's
effective plan when the accessor is operator or cohost, then passes
that through the dispatcher to OperatorStreamInterface. An admin
operator sitting on a Pro host's stream sees every premium card
unlocked; a Free-tier operator on a Free host's stream cannot flip
on premium cards the host couldn't keep.

What changes vs. owner
======================
1. NO useHostStream. Watch via useSimpleStream, never broadcast.
2. NO StageActions (Go Live / Pause / End / Restart / Recording).
   Replaced with an OperatorBadgeStrip in the same slot that shows
   role + live status + a Back button.
3. NO local mic / camera flip / data-saver controls. The operator
   has no local mediaStream.
4. Music deck = OperatorAudioPanel (existing relay-command surface),
   not the host's MusicDeck. Live audio mixing must happen on the
   owner's PC; the operator queues a command, the owner's stream-
   interface acts on it. Already-built code path; reused as-is.
5. HealthDeck rendered with isStreaming=false. The component already
   has a friendly "diagnostics appear once you go live" placeholder
   for that branch -- exactly right for an operator with no
   peer-connection visibility.
6. MediaDeck.onClipActiveChange = undefined. Operators can upload +
   play the host's short video clip, but auto-mic-mute is moot
   without a local mic. Caption + Mute-mic toggle still persist for
   the host to honour on their side.

Files changed
=============
- app/host/stream/[roomCode]/page.tsx
    - Resolve owner plan when access is operator/cohost.

- components/host/stream-interface.tsx
    - Dispatcher passes effectivePlan to OperatorStreamInterface.

- components/host/operator-stream-interface.tsx (rewrite)
    - 747 lines of legacy card-stack -> control-room layout.
    - Reuses ControlRoomTopbar, ProducerDeck, OverlayDeck,
      TickerDeck, MediaDeck (with VideoClipPanel), BrandingDeck,
      HealthDeck, ScenesRail, GuestsRail, CommsTabs, DirectorPanel,
      PrivateMessagesPanel, OperatorAudioPanel.
    - Camera switcher (DirectorPanel) calls
      /api/streams/{id}/active-participant POST so the owner's
      relayStream picks it up. Operator UI updates locally even if
      the API call fails (fail-soft).
    - New operator-mode banner above the grid explaining the role
      and that only the owner can start/end the broadcast.
